How it works

Two async precompile steps · one transaction each · Scheduler bridges the gap.

1

Fetch data

Agent calls the HTTP precompile (0x0801) inside a TEE to pull activity signals. Pending payload is stored; status becomes DataFetched.

2

Schedule analyze

Same transaction registers a one-shot Scheduler job. Next blocks, the chain itself invokes analyzeScore — no bot, no second click required.

3

LLM + SBT

LLM precompile (0x0802) scores 0–999 with a risk tier and short rationale. Core mints or updates a soulbound credential anyone can verify.

Owner-toggleable mock path generates deterministic scores without live precompiles — ideal when TEE executors flap during presentations.
