How it works

Two async precompile steps · one transaction each · Scheduler bridges the gap.

1

Fetch data

Agent calls the HTTP precompile (0x0801) inside a TEE to pull activity signals. Pending payload is stored; status becomes DataFetched.

2

Schedule analyze

Same transaction registers a one-shot Scheduler job. Next blocks, the chain itself invokes analyzeScore — no bot, no second click required.

3

LLM + SBT

LLM precompile (0x0802) scores 0–999 with a risk tier and short rationale. Core mints or updates a soulbound credential anyone can verify.

What is PrivaScore?

PrivaScore is an on-chain risk & trust scoring system built for Ritual Testnet (chain ID 1979). It turns wallet activity signals into a score from 0–999, a risk tier (Low / Medium / High), and a short reasoning string that anyone can verify.

How does the agent pipeline work?

The smart-contract agent is designed as a multi-step Ritual flow:

  1. Fetch — HTTP precompile 0x0801 pulls external data in a TEE.
  2. Schedule — Scheduler system contract queues the next step (no keeper bots).
  3. Analyze — LLM precompile 0x0802 (model zai-org/GLM-4.7-FP8) returns score + tier + reasoning.
  4. Settle — PrivaScoreCore records the result and mints/updates a soulbound SBT.

Ritual allows only one short-running async precompile per transaction, so HTTP and LLM are never called in the same tx.

Do I need MetaMask or a transfer to check a score?

No. On the dashboard, paste any wallet address and click Check Score. Lookup is fully read-only over https://rpc.ritualfoundation.org. If you never connect a wallet, the app never sends a transaction and never spends gas.

What is Mock Mode?

Mock Mode is an owner-toggle on the agent contract. When enabled, HTTP/LLM precompiles are simulated on-chain with deterministic data so demos stay stable if TEE executors are flaky. Production demos can keep Mock Mode on as a safety net while still using real contracts, events, and SBT minting.

What is the soulbound credential (SBT)?

Each settled address can receive a non-transferable ERC-721 credential from PrivaScoreCore. Transfers, approvals, and sales are blocked. Metadata is returned via on-chain tokenURI (base64 JSON) so explorers can show score and tier without an off-chain metadata server.

What do the score and risk tiers mean?
  • Score: integer 0–999 (higher = stronger trust signals in the demo model).
  • Tier 0 · Low: typically score < 333 — healthier activity profile.
  • Tier 1 · Medium: 333–665 — mixed behavioral signals.
  • Tier 2 · High: ≥ 666 — elevated anomaly / short-tenure risk.

Tier thresholds are applied on-chain when parsing LLM or mock output. Reasoning is a short human-readable summary stored with the score record.

How do I use the dashboard?
  1. Open the app dashboard.
  2. Paste a wallet address, or click Use on the leaderboard.
  3. Click Check Score (or press Enter).
  4. Watch the 1 → 2 → 3 pipeline animation, then read the score, tier, and reasoning.

Leaderboard mixes live on-chain settlements with sample rows for a fuller demo board. No wallet connection is required for lookup.

Any Ritual-specific gotchas we handled?
  • Priority fee must be ≥ 1 gwei or txs can vanish silently.
  • Do not use --legacy transactions on Ritual.
  • block.timestamp is in milliseconds on Ritual.
  • Precompile mocks must use fallback (avoid double ABI-encoding).
  • LLM requests must include convoHistory (empty tuple is valid).
  • AsyncDelivery callbacks must verify msg.sender == AsyncDelivery.
